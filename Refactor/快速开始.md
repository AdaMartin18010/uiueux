## 📚 深入学习

### 按技术领域学习

#### 1️⃣ 终端类型

了解不同平台的特性和开发要求：

- **Web端**：浏览器环境、Web标准、响应式设计
- **移动端**：原生开发、跨端框架、性能优化
- **桌面端**：桌面应用、跨平台、用户体验
- **游戏端**：游戏引擎、图形渲染、交互设计

#### 2️⃣ 技术栈与框架

掌握主流技术栈和框架：

- **前端框架**：React、Vue、Angular、新兴框架
- **跨端技术**：Flutter、React Native、Tauri
- **工程化工具**：构建工具、包管理、CI/CD
- **Web核心技术**：HTML、CSS、JavaScript、Web API

#### 3️⃣ 编程语言范式

学习不同编程范式和语言特性：

- **系统编程**：Rust、C++、性能优化
- **函数式编程**：Haskell、Scala、纯函数
- **Web开发**：TypeScript、JavaScript、现代语法
- **移动开发**：Dart、Flutter、跨平台

#### 4️⃣ 设计模式与架构

掌握软件设计和架构模式：

- **GoF设计模式**：23种经典设计模式
- **架构模式**：MVC、MVVM、MVP、Flux、Redux、微服务、事件驱动（详见[4.3 组件化与架构模式](4.设计模式与架构/4.3 组件化与架构模式.md)）
- **组件设计**：组件化、模块化、可复用性
- **认知分析**：设计思维、用户体验、认知科学

#### 5️⃣ 技术规范与标准

遵循开发规范和最佳实践：

- **设计规范**：UI/UX设计、可访问性、国际化
- **开发规范**：代码规范、命名约定、注释标准
- **性能优化**：加载优化、运行时优化、监控
- **质量保证**：测试、代码审查、持续集成

#### 6️⃣ 人工智能原理与算法

了解AI技术和应用：

- **AI基础**：机器学习、深度学习、神经网络
- **经典算法**：监督学习、无监督学习、强化学习
- **大模型**：Transformer、GPT、BERT、应用实践
- **AI工程**：模型部署、监控、伦理、治理

## 🛠️ 工具使用

### 自动化工具

#### 运行工具

```bash
# 进入工具目录
cd Refactor/tools

# 运行所有工具（Python版本）
python generate_toc.py

# 运行特定工具
python generate_toc.py toc      # 生成目录
python generate_toc.py readme   # 更新README
python generate_toc.py links    # 检查链接
python generate_toc.py refs     # 生成交叉引用
```

#### Windows用户

```cmd
# 批处理脚本
generate-toc.bat

# PowerShell脚本
.\generate-toc.ps1
```

### 贡献指南

#### 内容贡献

1. **Fork项目**：复制项目到你的账户
2. **创建分支**：`git checkout -b feature/your-feature`
3. **编辑内容**：修改或添加文档
4. **提交更改**：`git commit -m "feat: 添加新内容"`
5. **推送分支**：`git push origin feature/your-feature`
6. **创建PR**：提交Pull Request

#### 格式规范

- **文件命名**：`编号. 专题名称.md`
- **标题层级**：使用 `#` 到 `#####`
- **代码块**：指定语言类型
- **链接格式**：`[显示文本](链接地址)`
